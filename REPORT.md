## 1. Введение

Настоящий отчёт содержит описание шести лабораторных работ, выполненных в рамках курса «Системы передачи и защиты информации». Все работы оформлены в соответствии с моделью алфавитной криптосистемы, предложенной В.О. Осипяном:

$$\Sigma_0 = (M^{\ast}, Q, C^{\ast}, E(m), D(c) \mid V(E(m), D(c)))$$

где:
- $M^{\ast}$ — множество всех открытых текстов (сообщений);
- $Q$ — алфавит (множество числовых эквивалентов элементарных сообщений);
- $C^{\ast}$ — множество шифротекстов (кодовых слов);
- $E(m)$ — алгоритм кодирования (шифрования);
- $D(c)$ — алгоритм декодирования (дешифрования);
- $V(E(m), D(c))$ — верификация: $D(E(m)) = m$ для любого $m \in M^{\ast}$.

Работы разделены на два тематических блока:

**Помехоустойчивое кодирование** (задачи 1–3): коды, исправляющие ошибки при передаче данных по каналам связи — код Хэмминга, расширенный код Хэмминга, код Варшамова-Тененгольца.

**Криптосистемы с открытым ключом** (задачи 4–6): асимметричные криптосистемы, безопасность которых основана на NP-сложных задачах — RSA, аддитивный рюкзак (АВКР), мультипликативный рюкзак (МВКР).

Все алгоритмы реализованы на языке Python 3.12. Задачи 1–3 работают в консольном режиме. Задачи 4–6 дополнительно имеют графический интерфейс на PyQt6. Управление всеми задачами осуществляется через единое CLI-приложение `main.py`.

## 2. Задача 1. Код Хэмминга C(n, k)

### 2.1. Постановка задачи

Реализовать передачу данных по каналу связи с **обнаружением и исправлением одной симметричной ошибки** с использованием кода Хэмминга.

### 2.2. Идея метода

Код Хэмминга — линейный блочный код, добавляющий к $k$ информационным битам $r$ проверочных бит, образуя кодовое слово длины $n = k + r$. Число проверочных бит определяется условием:

$$2^r \geq k + r + 1$$

Проверочные биты размещаются на позициях, являющихся степенями двойки ($2^0, 2^1, \ldots, 2^{r-1}$). Каждый проверочный бит $p_j$ вычисляется как XOR бит на позициях, в двоичном представлении которых $j$-й бит равен 1.

### 2.3. Алгоритм

**Кодирование $E(m)$:**
1. Информационные биты размещаются на не-степенных позициях.
2. Вычисляются проверочные биты: $p_j = \bigoplus_{\text{бит } j \text{ в } i = 1} c_i$.

**Декодирование $D(c)$:**
1. Вычисляется синдром $S = (s_{r-1}, \ldots, s_0)_2$.
2. Если $S = 0$ — ошибок нет. Если $S \neq 0$ — ошибка на позиции $S$, инвертируем бит.
3. Извлекаем информационные биты.

### 2.4. Пример работы

Код $C(7, 4)$: $k = 4$, $r = 3$, $n = 7$.

| Позиция | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|
| Тип | $p_0$ | $p_1$ | $m_1$ | $p_2$ | $m_2$ | $m_3$ | $m_4$ |

Сообщение $m = (1, 0, 1, 1)$ → кодовое слово $c = E(m) = (0, 1, 1, 0, 0, 1, 1)$.

При ошибке на позиции 5: $c' = (0, 1, 1, 0, 1, 1, 1)$ → синдром $S = 5$ → исправляем → $m = (1, 0, 1, 1)$ ✓

### 2.5. Реализация

Реализован класс `HammingCodeSystem` с методами `encode()`, `decode()`, `verify()`. Файл: `task_1/hamming_code.py`.

### 2.6. Связь кода с теорией

| Формула | Код | Пояснение |
|---|---|---|
| $2^r \geq k + r + 1$ | `while (1 << r) < k + r + 1` | Минимальное число проверочных бит |
| $p_j = \bigoplus c_i$ | `parity ^= codeword[i]` | Вычисление проверочного бита через XOR |
| Синдром $S$ | `syndrome \|= parity_pos` | Двоичный номер ошибочной позиции |
| $D(E(m)) = m$ | `decoded == message` | Верификация криптосистемы |

## 3. Задача 2. Расширенный код Хэмминга C[n+1, k]

### 3.1. Постановка задачи

Реализовать расширенный код Хэмминга, который **исправляет одну ошибку** и **обнаруживает две ошибки**.

### 3.2. Идея метода

К стандартному коду Хэмминга $C(n, k)$ добавляется один бит общей чётности $p_{\text{overall}} = \bigoplus_{i=1}^{n} c_i$. Итоговая длина — $n_{\text{ext}} = n + 1$.

Минимальное расстояние Хэмминга увеличивается с $d_{\min} = 3$ до $d_{\min} = 4$, что позволяет:
- исправлять 1 ошибку: $t = \lfloor(4-1)/2\rfloor = 1$;
- обнаруживать 2 ошибки: $d_{\min} - 1 = 3 \geq 2$.

### 3.3. Таблица решений

| Синдром $S$ | Общая чётность $P$ | Интерпретация |
|---|---|---|
| $S = 0$ | $P = 0$ | Ошибок нет |
| $S = 0$ | $P = 1$ | Ошибка в $p_{\text{overall}}$ (данные корректны) |
| $S \neq 0$ | $P = 1$ | Одиночная ошибка на позиции $S$ — **исправляем** |
| $S \neq 0$ | $P = 0$ | Двойная ошибка — **обнаружена**, не исправляема |

### 3.4. Пример работы

Код $C_{[8, 4]}$: $k = 4$, $r = 3$, $n = 7$, $n_{\text{ext}} = 8$.

Сообщение $m = (1, 0, 1, 1)$ → расширенное слово $c = (0, 0, 1, 1, 0, 0, 1, 1)$.

- Одиночная ошибка (позиция 5): $S = 5$, $P = 1$ → исправляем ✓
- Двойная ошибка (позиции 2, 5): $S = 7$, $P = 0$ → обнаружена ✓

### 3.5. Реализация

Реализован класс `ExtendedHammingCodeSystem` с методами `encode()`, `decode()`, `verify()`. Файл: `task_2/hamming_extended.py`.

## 4. Задача 3. Код Варшамова-Тененгольца VT_a(n)

### 4.1. Постановка задачи

Реализовать код Варшамова-Тененгольца для **исправления одной асимметричной ошибки** в Z-канале.

### 4.2. Идея метода

В отличие от кода Хэмминга, который работает в симметричном канале (BSC), код VT предназначен для **асимметричного канала** (Z-канала), где ошибки происходят только в одном направлении: $1 \to 0$ или $0 \to 1$.

Код определяется как:

$$VT_a(n) = \lbrace x \in \lbrace 0,1 \rbrace^{n} \mid \sum_{i=1}^{n} i \cdot x_i \equiv a \pmod{n+1} \rbrace$$

### 4.3. Синдром Варшамова

Для слова $y$ синдром:

$$S(y) = \sum_{i=1}^{n} i \cdot y_i \pmod{n+1}$$

Если $S(y) = a$ — ошибок нет. Иначе позиция ошибки:
- Z-канал ($1 \to 0$): $j = (a - S(y)) \bmod{(n+1)}$
- Обратный Z-канал ($0 \to 1$): $j = (S(y) - a) \bmod{(n+1)}$

### 4.4. Кодирование

Используется **табличное кодирование**: генерируется кодовая книга $C^{\ast} = VT_a(n)$, сообщение $m$ интерпретируется как индекс в кодовой книге.

Мощность кода: $|VT_a(n)| \approx 2^n / (n+1)$.

Скорость кода: $R = k/n \approx 1 - \log_2(n+1)/n \to 1$ при $n \to \infty$ (асимптотически оптимальный).

### 4.5. Пример работы

Код $VT_0(7)$: $n = 7$, $a = 0$, $k = 4$, $|C^{\ast}| = 16$.

Сообщение $m = (1, 0, 1, 1)$ → $c = (1, 0, 1, 1, 0, 0, 0)$.

Ошибка $1 \to 0$ на позиции 4: $S(y) = 4$, $j = (0 - 4) \bmod 8 = 4$ → исправляем ✓

### 4.6. Реализация

Реализован класс `VarshamovCodeSystem` с методами `encode()`, `decode()`, `verify()`, `introduce_error()`. Файл: `task_3/varshamov_code.py`.

## 5. Задача 4. Алгоритм RSA

### 5.1. Постановка задачи

Реализовать асимметричную криптосистему RSA с открытым ключом, основанную на **NP-сложной задаче факторизации целых чисел**.

### 5.2. Идея метода

RSA — криптосистема с открытым ключом, в которой шифрование и дешифрование выполняются разными ключами:

- **Открытый ключ** $(e, n)$: используется для шифрования.
- **Закрытый ключ** $(d, n)$: используется для дешифрования.

Безопасность основана на том, что зная $n = p \cdot q$, вычислительно невозможно найти $p$ и $q$ за полиномиальное время.

### 5.3. Генерация ключей

1. Выбираются два различных простых числа $p$ и $q$.
2. Вычисляется модуль: $n = p \cdot q$.
3. Вычисляется функция Эйлера: $\varphi(n) = (p-1)(q-1)$.
4. Выбирается открытая экспонента: $1 < e < \varphi(n)$, $\gcd(e, \varphi(n)) = 1$.
5. Вычисляется секретная экспонента: $d = e^{-1} \bmod \varphi(n)$ (расширенный алгоритм Евклида).

### 5.4. Шифрование и дешифрование

$$E(m) = m^e \bmod n, \qquad D(c) = c^d \bmod n$$

### 5.5. Доказательство корректности

По теореме Эйлера: $m^{\varphi(n)} \equiv 1 \pmod{n}$ при $\gcd(m, n) = 1$.

Поскольку $e \cdot d = 1 + k \cdot \varphi(n)$:

$$D(E(m)) = m^{e \cdot d} = m^{1 + k\varphi(n)} = m \cdot (m^{\varphi(n)})^k \equiv m \pmod{n}$$

### 5.6. NP-сложная задача факторизации

Наилучший классический алгоритм — решето числового поля (GNFS):

$$T(n) = \exp \bigl( c \cdot (\ln n)^{1/3} \cdot (\ln \ln n)^{2/3} \bigr), \quad c \approx 1.923$$

| Размер ключа (бит) | Уровень безопасности | Статус |
|---|---|---|
| 512 | ~30 бит | Взломан (1999) |
| 1024 | ~80 бит | Не рекомендуется |
| 2048 | ~112 бит | Безопасен до ~2030 г. |
| 4096 | ~140 бит | Высокий уровень |

### 5.7. Реализация

Реализован класс `RSACryptoSystem` с методами `encrypt()`, `decrypt()`, `verify()`, `encrypt_text()`, `decrypt_text()`. GUI на PyQt6 с 4 вкладками: генерация ключей, шифрование чисел, шифрование текста, демонстрация факторизации. Файл: `task_4/rsa_crypto.py`.

## 6. Задача 5. Аддитивный рюкзак (АВКР)

### 6.1. Постановка задачи

Реализовать криптосистему Меркла-Хеллмана, основанную на **NP-полной задаче о сумме подмножеств** (Subset Sum Problem).

### 6.2. Идея метода

Схема основана на преобразовании «лёгкого» рюкзака (сверхвозрастающая последовательность) в «трудный» (произвольная последовательность) с помощью модулярной арифметики.

### 6.3. Генерация ключей

1. **Закрытый ключ:** сверхвозрастающая последовательность $B = (b_1, \ldots, b_n)$, где $b_i > \sum_{j=1}^{i-1} b_j$.
2. Выбирается модуль $q > \sum b_i$ (простое число).
3. Выбирается множитель $w$: $\gcd(w, q) = 1$.
4. **Открытый ключ:** $A = (a_1, \ldots, a_n)$, где $a_i = w \cdot b_i \bmod q$.

### 6.4. Шифрование и дешифрование

**Шифрование:** $c = E(m) = \sum_{i=1}^{n} m_i \cdot a_i$

**Дешифрование:**
1. $S' = c \cdot w^{-1} \bmod q$ (переход к «лёгкому» рюкзаку).
2. Жадный алгоритм по $B$: если $b_i \leq S'$, то $m_i = 1$, $S' = S' - b_i$.

Корректность: $S' = c \cdot w^{-1} = (\sum m_i \cdot w \cdot b_i) \cdot w^{-1} = \sum m_i \cdot b_i \pmod{q}$.

### 6.5. NP-сложная задача

Задача о сумме подмножеств (NP-полна, доказано Р. Карпом, 1972):
- Полный перебор: $O(2^n)$
- Meet-in-the-middle: $O(2^{n/2})$
- Динамическое программирование: $O(n \cdot S)$ (псевдополиномиальный)

**Замечание:** в 1982 г. А. Шамир показал, что оригинальная схема Меркла-Хеллмана может быть взломана за полиномиальное время с помощью LLL-алгоритма.

### 6.6. Пример работы

$B = (2, 5, 11, 23, 47, 95, 191, 383)$, $q = 881$, $w = 588$.

Сообщение $m = (1, 0, 1, 1, 0, 0, 1, 0)$:
1. $c = \sum m_i \cdot a_i$ (сумма по открытому ключу).
2. $S' = c \cdot w^{-1} \bmod 881$.
3. Жадный алгоритм по $B$ → $m = (1, 0, 1, 1, 0, 0, 1, 0)$ ✓

### 6.7. Реализация

Реализован класс `KnapsackCryptoSystem` с методами `encrypt_block()`, `decrypt_block()`, `verify_block()`, `encrypt_text()`, `decrypt_text()`. GUI на PyQt6 с 4 вкладками. Файл: `task_5/knapsack_crypto.py`.

## 7. Задача 6. Мультипликативный рюкзак (МВКР)

### 7.1. Постановка задачи

Реализовать криптосистему мультипликативного рюкзака, основанную на **NP-полной задаче о произведении подмножеств** (Subset Product Problem).

### 7.2. Идея метода

Аналогична АВКР, но вместо суммы используется произведение, а вместо модулярного умножения — модулярное возведение в степень.

### 7.3. Генерация ключей

1. **Закрытый ключ:** набор различных простых чисел $B = (p_1, p_2, \ldots, p_n)$.
2. Модуль $q > \prod p_i$ (простое число).
3. Секретный показатель $w$: $\gcd(w, q-1) = 1$ (обратите внимание: $q-1$, а не $q$!).
4. **Открытый ключ:** $A = (a_1, \ldots, a_n)$, где $a_i = p_i^w \bmod q$.

### 7.4. Шифрование и дешифрование

**Шифрование:** $c = E(m) = \prod_{i=1}^{n} a_i^{m_i} \bmod q$

**Дешифрование:**
1. $P = c^{w^{-1}} \bmod q$, где $w^{-1}$ по модулю $q - 1$ (малая теорема Ферма).
2. Факторизация $P$ по известным простым $B$: если $P \div p_i$ нацело, то $m_i = 1$.

Корректность: $((\prod p_i^{w \cdot m_i}))^{w^{-1}} = \prod p_i^{m_i \cdot w \cdot w^{-1}} = \prod p_i^{m_i} \pmod{q}$.

### 7.5. Отличия от АВКР

| Характеристика | АВКР (аддитивный) | МВКР (мультипликативный) |
|---|---|---|
| Операция шифрования | Сумма: $\sum m_i \cdot a_i$ | Произведение: $\prod a_i^{m_i} \bmod q$ |
| Закрытый ключ | Сверхвозрастающая последовательность | Различные простые числа |
| Маскировка | $a_i = w \cdot b_i \bmod q$ | $a_i = p_i^w \bmod q$ |
| Обратное преобразование | $S' = c \cdot w^{-1} \bmod q$ | $P = c^{w^{-1}} \bmod q$ |
| Решение лёгкого рюкзака | Жадный алгоритм | Факторизация по простым |
| Модуль обратного | $w^{-1} \bmod q$ | $w^{-1} \bmod (q-1)$ |
| Трудная задача | Subset Sum (NP-полная) | Subset Product + DLP |

### 7.6. Реализация

Реализован класс `MultiplicativeKnapsackCryptoSystem` с методами `encrypt_block()`, `decrypt_block()`, `verify_block()`. GUI на PyQt6 с 4 вкладками. Файл: `task_6/multiplicative_knapsack_crypto.py`.

| $Q$ | $\{0, 1\}$ | $\{0, 1, \ldots, n-1\}$ / $\{0, 1\}$ |
| $C^{\ast}$ | Кодовые слова длины $n$ | Шифротекст (целые числа) |
| $E(m)$ | Добавление избыточности | Модулярная арифметика |
| $D(c)$ | Синдромное декодирование | Обратная модулярная операция |
| $V$ | $D(E(m)) = m$ | $D(E(m)) = m$ |

## 9. Заключение

В ходе выполнения лабораторных работ были реализованы и исследованы:

1. **Код Хэмминга C(n, k)** — линейный блочный код, исправляющий одну симметричную ошибку. Реализованы кодирование, синдромное декодирование с исправлением, верификация.

2. **Расширенный код Хэмминга C[n+1, k]** — модификация стандартного кода с дополнительным битом общей чётности, повышающим минимальное расстояние до $d_{\min} = 4$, что позволяет обнаруживать двойные ошибки.

3. **Код Варшамова-Тененгольца VT_a(n)** — нелинейный код для асимметричных каналов (Z-каналов). Реализовано табличное кодирование, декодирование с использованием синдрома Варшамова, имитация Z-канала.

4. **Алгоритм RSA** — асимметричная криптосистема, основанная на NP-сложности задачи факторизации. Реализованы генерация ключей, шифрование/дешифрование чисел и текста, демонстрация факторизации. GUI с 4 вкладками.

5. **Аддитивный рюкзак (АВКР)** — криптосистема Меркла-Хеллмана, основанная на NP-полной задаче о сумме подмножеств. Реализованы блочное и текстовое шифрование, демонстрация NP-сложности. GUI с 4 вкладками.

6. **Мультипликативный рюкзак (МВКР)** — мультипликативный аналог АВКР, использующий произведение вместо суммы и возведение в степень вместо умножения. Дополнительно связан с задачей дискретного логарифма. GUI с 4 вкладками.

Все реализации включают:
- Функцию кодирования/шифрования $E(m)$;
- Функцию декодирования/дешифрования $D(c)$;
- Верификацию $V(E(m), D(c))$: $D(E(m)) = m$;
- Подробные комментарии, связывающие код с математическими формулами;
- Валидацию входных данных.

Единое CLI-приложение `main.py` обеспечивает удобный доступ ко всем задачам через интерактивное меню.
