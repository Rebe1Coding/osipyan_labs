# Код Хэмминга $C(n, k)$ — передача данных с исправлением одиночной ошибки

## 1. Идея метода (схематично)

Код Хэмминга — это линейный блочный код, который позволяет **обнаружить и исправить одну** симметричную ошибку в двоичном слове при передаче по каналу связи.

Принцип работы:

```
Отправитель                     Канал связи                  Получатель
┌───────────┐                ┌──────────────┐             ┌──────────────┐
│ Сообщение │   E(m)         │  Возможная   │   D(c')     │ Вычисление   │
│ m (k бит) │ ──────────►    │  ошибка в    │ ──────────► │ синдрома S   │
│           │  Добавление    │  одном бите  │             │              │
│           │  r проверочных │              │             │ S=0 → нет    │
│           │  бит           │              │             │   ошибки     │
│           │  c = E(m)      │  c' = c ⊕ e  │             │ S≠0 → ошибка│
│           │  (n = k+r бит) │              │             │   на позиции │
│           │                │              │             │   S, инверт. │
└───────────┘                └──────────────┘             └──────────────┘
                                                                │
                                                                ▼
                                                         ┌──────────────┐
                                                         │ Извлечение   │
                                                         │ информац.    │
                                                         │ бит → m      │
                                                         └──────────────┘
```

В терминологии криптосистемы Осипяна задача оформляется в виде кортежа:

$$\Sigma_0 = (M^*, Q, C^*, E(m), D(c) \mid V(E(m), D(c)))$$

где:
- $M^*$ — множество всех открытых текстов (сообщений) длины $k$;
- $Q$ — алфавит $\{0, 1\}$;
- $C^*$ — множество кодовых слов (шифротекстов) длины $n$;
- $E(m)$ — алгоритм кодирования (шифрования): $m \to c$;
- $D(c)$ — алгоритм декодирования (дешифрования): $c \to m$;
- $V(E(m), D(c))$ — верификация: $D(E(m)) = m$ для любого $m \in M^*$.

Алгоритмы $E(m)$ и $D(c)$ связаны тем, что для любого сообщения $m = m_1 m_2 \ldots m_k \in M^*$ однозначно преобразуется в кодовое слово $c = c_1 c_2 \ldots c_n \in C^*$, и обратно: по кодовому слову $c$ всегда можно восстановить исходное сообщение $m$.

## 2. Математическое описание

### 2.1. Параметры кода

Код Хэмминга $C(n, k)$ определяется числом информационных бит $k$ и числом проверочных бит $r$, связанных соотношением:

$$2^r \geq k + r + 1$$

Длина кодового слова:

$$n = k + r$$

Для классических кодов Хэмминга $n = 2^r - 1$, $k = 2^r - 1 - r$.

### 2.2. Кодирование $E(m)$

Дано сообщение $m = (m_1, m_2, \ldots, m_k)$, где $m_i \in \{0, 1\}$.

**Шаг 1.** Информационные биты размещаются на позициях кодового слова, **не являющихся** степенями двойки:

$$\text{Позиции информационных бит: } i \in \{1, 2, \ldots, n\} \setminus \{2^0, 2^1, \ldots, 2^{r-1}\}$$

**Шаг 2.** Проверочные биты $p_j$ размещаются на позициях $2^j$ ($j = 0, 1, \ldots, r-1$) и вычисляются как:

$$p_j = \bigoplus_{\substack{i=1 \\ \text{бит } j \text{ в } i = 1}}^{n} c_i, \quad j = 0, 1, \ldots, r-1$$

где $\oplus$ — операция XOR (сложение по модулю 2), а сумма берётся по всем позициям $i$, в двоичном представлении которых $j$-й бит равен 1 (кроме самой позиции $2^j$).

### 2.3. Декодирование $D(c)$

Принято слово $c' = (c'_1, c'_2, \ldots, c'_n)$, возможно, содержащее одну ошибку.

**Шаг 1.** Вычисляется **синдром** $S = (s_{r-1}, \ldots, s_1, s_0)_2$:

$$s_j = \bigoplus_{\substack{i=1 \\ \text{бит } j \text{ в } i = 1}}^{n} c'_i, \quad j = 0, 1, \ldots, r-1$$

**Шаг 2.** Интерпретация синдрома:
- Если $S = 0$ — ошибок нет.
- Если $S \neq 0$ — ошибка на позиции $S$; инвертируем бит: $c'_S := c'_S \oplus 1$.

**Шаг 3.** Извлекаем информационные биты из исправленного слова (позиции, не являющиеся степенями 2).

### 2.4. Верификация $V(E(m), D(c))$

Свойство корректности криптосистемы:

$$\forall m \in M^*: \quad D(E(m)) = m$$

## 3. Реализация (код)

```python
"""
Код Хэмминга C(n, k) — передача данных по каналу связи
с обнаружением и исправлением одной симметричной ошибки.

Оформлено в соответствии с моделью алфавитной криптосистемы
В.О. Осипяна: Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))).

Автор алгоритма: В.О. Осипян
"""

from __future__ import annotations

import math
from dataclasses import dataclass


# ---------------------------------------------------------------------------
#  Вспомогательные функции
# ---------------------------------------------------------------------------

def _calc_parity_bits(k: int) -> int:
    """Вычисляет минимальное число проверочных бит r для k информационных.

    Условие Хэмминга: 2^r >= k + r + 1  (т.е. n = k + r, n+1 <= 2^r).
    """
    # Перебираем r начиная с 0, пока не выполнится неравенство.
    # Это прямая реализация условия из теории: минимальное r,
    # при котором 2^r покрывает все n+1 возможных синдромов
    # (n позиций + «ошибки нет»).
    r = 0
    while (1 << r) < k + r + 1:
        r += 1
    return r


def _is_power_of_two(x: int) -> bool:
    """Проверяет, является ли x степенью двойки (позиция проверочного бита).

    Битовый трюк: у степеней двойки ровно один установленный бит,
    поэтому x & (x-1) == 0.
    """
    return x > 0 and (x & (x - 1)) == 0


# ---------------------------------------------------------------------------
#  Класс-криптосистема Хэмминга
# ---------------------------------------------------------------------------

@dataclass
class HammingCodeSystem:
    """Реализация криптосистемы Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c)))
    на основе кода Хэмминга C(n, k).

    Атрибуты
    ---------
    k : int
        Число информационных бит (размерность кода).
    r : int
        Число проверочных (контрольных) бит.
    n : int
        Длина кодового слова (n = k + r).

    В терминологии Осипяна
    ----------------------
    M*  — множество всех сообщений (двоичных строк длины k).
    Q   — множество числовых эквивалентов элементарных сообщений {0, 1}.
    C*  — множество шифротекстов (кодовых слов длины n).
    E(m) — алгоритм кодирования (шифрования): m -> c.
    D(c) — алгоритм декодирования (дешифрования): c -> m.
    V(E(m), D(c)) — верификация: D(E(m)) == m для любого m ∈ M*.
    """

    k: int  # число информационных бит
    r: int = 0  # число проверочных бит (вычисляется автоматически)
    n: int = 0  # длина кодового слова

    def __post_init__(self) -> None:
        if self.k < 1:
            raise ValueError(f"k должно быть >= 1, получено {self.k}")
        self.r = _calc_parity_bits(self.k)
        self.n = self.k + self.r

    # ---- E(m): кодирование (шифрование) ---------------------------------

    def encode(self, message: list[int]) -> list[int]:
        """Кодирует информационное слово m длины k в кодовое слово c длины n.

        Это функция E(m) криптосистемы — прямое преобразование.
        """
        self._validate_message(message)

        # Создаём массив длины n+1 (индексация с 1, как в теории Хэмминга).
        # Позиция 0 не используется — это позволяет напрямую соотносить
        # номера позиций с двоичным представлением, как в формулах.
        codeword = [0] * (self.n + 1)

        # Шаг 1: расставляем информационные биты на НЕ-степенные позиции.
        # Степени двойки (1, 2, 4, 8, ...) зарезервированы под проверочные биты.
        # Это стандартное размещение по Хэммингу, которое обеспечивает
        # возможность вычислить синдром при декодировании.
        j = 0
        for i in range(1, self.n + 1):
            if not _is_power_of_two(i):
                codeword[i] = message[j]
                j += 1

        # Шаг 2: вычисляем каждый проверочный бит.
        # p_j (на позиции 2^j) = XOR всех codeword[i],
        # где j-й бит числа i равен 1.
        # Это реализация формулы: p_j = ⊕ c_i (по i, у которых бит j = 1).
        for j in range(self.r):
            parity_pos = 1 << j          # позиция 2^j
            parity = 0
            for i in range(1, self.n + 1):
                if i & parity_pos and i != parity_pos:
                    # i & parity_pos проверяет, что j-й бит позиции i = 1
                    # i != parity_pos исключает саму позицию проверочного бита
                    parity ^= codeword[i]
            codeword[parity_pos] = parity

        return codeword[1:]

    # ---- D(c): декодирование (дешифрование) ------------------------------

    def decode(self, received: list[int]) -> tuple[list[int], int]:
        """Декодирует принятое слово c длины n, исправляя одиночную ошибку.

        Это функция D(c) криптосистемы — обратное преобразование.

        Возвращает (decoded_message, error_position).
        error_position = 0 означает отсутствие ошибки.
        """
        self._validate_codeword(received)

        # Переходим к индексации с 1
        cw = [0] + list(received)

        # Шаг 1: вычисляем синдром S.
        # s_j = XOR всех cw[i], где j-й бит позиции i = 1.
        # В отличие от кодирования, здесь сумма включает и саму
        # позицию проверочного бита — это позволяет обнаружить ошибку
        # как в информационных, так и в проверочных битах.
        syndrome = 0
        for j in range(self.r):
            parity_pos = 1 << j
            s_j = 0
            for i in range(1, self.n + 1):
                if i & parity_pos:
                    s_j ^= cw[i]
            if s_j:
                syndrome |= parity_pos

        # Шаг 2: если синдром ≠ 0, он указывает на позицию ошибки.
        # Это ключевое свойство кода Хэмминга: синдром — это
        # двоичный номер ошибочной позиции.
        error_position = syndrome
        if error_position != 0:
            if error_position > self.n:
                raise ValueError(
                    f"Синдром {error_position} выходит за пределы "
                    f"кодового слова длины {self.n}. "
                    "Возможно, произошло более одной ошибки."
                )
            cw[error_position] ^= 1  # инвертируем ошибочный бит

        # Шаг 3: извлекаем информационные биты из исправленного слова
        decoded = []
        for i in range(1, self.n + 1):
            if not _is_power_of_two(i):
                decoded.append(cw[i])

        return decoded, error_position

    # ---- V(E(m), D(c)): верификация --------------------------------------

    def verify(self, message: list[int]) -> bool:
        """Проверяет свойство криптосистемы: D(E(m)) == m для данного m."""
        self._validate_message(message)
        encoded = self.encode(message)
        decoded, _ = self.decode(encoded)
        return decoded == message

    # ---- Валидация -------------------------------------------------------

    def _validate_message(self, message: list[int]) -> None:
        if len(message) != self.k:
            raise ValueError(
                f"Длина сообщения должна быть {self.k}, получено {len(message)}"
            )
        if not all(b in (0, 1) for b in message):
            raise ValueError("Сообщение должно содержать только 0 и 1")

    def _validate_codeword(self, codeword: list[int]) -> None:
        if len(codeword) != self.n:
            raise ValueError(
                f"Длина кодового слова должна быть {self.n}, "
                f"получено {len(codeword)}"
            )
        if not all(b in (0, 1) for b in codeword):
            raise ValueError("Кодовое слово должно содержать только 0 и 1")
```

## 4. Связь кода с теорией

| Математическая формула | Строка кода | Пояснение |
|---|---|---|
| $2^r \geq k + r + 1$ | `while (1 << r) < k + r + 1` | Вычисление минимального $r$ — количества проверочных бит. `1 << r` — это $2^r$ в битовой арифметике. |
| Позиции проверочных бит: $2^0, 2^1, \ldots, 2^{r-1}$ | `_is_power_of_two(i)` | Отделяем позиции для проверочных бит от информационных. Степень двойки определяется битовым трюком `x & (x-1) == 0`. |
| $p_j = \bigoplus c_i$ (бит $j$ в $i = 1$) | `parity ^= codeword[i]` внутри `if i & parity_pos` | Вычисление проверочного бита $p_j$. Условие `i & parity_pos` проверяет, что $j$-й бит позиции $i$ равен 1, как требует формула. `^=` — это XOR ($\oplus$). |
| Синдром $s_j = \bigoplus c'_i$ | `s_j ^= cw[i]` внутри `if i & parity_pos` | Вычисление $j$-го бита синдрома. При декодировании суммируем **все** позиции (включая проверочные), чтобы обнаружить любую ошибку. |
| $S \neq 0 \Rightarrow$ ошибка на позиции $S$ | `cw[error_position] ^= 1` | Синдром в двоичном виде даёт номер ошибочной позиции. Инвертируем этот бит для исправления. |
| $D(E(m)) = m$ | `decoded == message` в `verify()` | Прямая проверка свойства верификации $V(E(m), D(c))$ криптосистемы. |

## 5. Пример работы

### Код Хэмминга $C(7, 4)$: $k = 4$, $r = 3$, $n = 7$

**Кодирование** сообщения $m = (1, 0, 1, 1)$:

Позиции кодового слова (нумерация с 1):

| Позиция | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|
| Тип | $p_0$ | $p_1$ | $m_1$ | $p_2$ | $m_2$ | $m_3$ | $m_4$ |
| Значение | 0 | 1 | **1** | 1 | **0** | **1** | **1** |

Кодовое слово: $c = E(m) = (0, 1, 1, 0, 0, 1, 1)$

**Декодирование** с ошибкой на позиции 5:

Принято: $c' = (0, 1, 1, 0, \mathbf{1}, 1, 1)$

Синдром: $S = (s_2, s_1, s_0) = (1, 0, 1)_2 = 5$ → ошибка на позиции 5. Исправляем, получаем $m = (1, 0, 1, 1)$ ✓
