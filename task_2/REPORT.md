# Расширенный код Хэмминга $C_{[n+1, k]}$ — передача данных с исправлением одиночной и обнаружением двойной ошибки

## 1. Идея метода (схематично)

Расширенный код Хэмминга — это модификация стандартного кода Хэмминга $C(n, k)$, в котором к кодовому слову добавляется **один дополнительный бит общей чётности** $p_{\text{overall}}$. Итоговая длина кодового слова — $n + 1$.

Это расширение позволяет:
- **исправить 1 ошибку** (как и стандартный код);
- **обнаружить 2 ошибки** (стандартный код этого не может).

Принцип работы:

```
Отправитель                     Канал связи                  Получатель
┌───────────┐                ┌──────────────┐             ┌──────────────────┐
│ Сообщение │   E(m)         │  Возможные   │   D(c')     │ 1. Синдром S     │
│ m (k бит) │ ──────────►    │  ошибки      │ ──────────► │ 2. Общая чётн. P │
│           │  + r пров.     │  (1 или 2    │             │                  │
│           │  + 1 бит       │   бита)      │             │ S=0, P=0 → OK   │
│           │  общей чётн.   │              │             │ S=0, P=1 → ошиб.│
│           │                │  c' = c ⊕ e  │             │   в p_overall    │
│           │  c = E(m)      │              │             │ S≠0, P=1 → 1 ош.│
│           │  (n+1 бит)     │              │             │   исправляем     │
│           │                │              │             │ S≠0, P=0 → 2 ош.│
│           │                │              │             │   обнаружены     │
└───────────┘                └──────────────┘             └──────────────────┘
                                                                │
                                                                ▼
                                                         ┌──────────────┐
                                                         │ Извлечение   │
                                                         │ информац.    │
                                                         │ бит → m      │
                                                         └──────────────┘
```

В терминологии криптосистемы Осипяна задача оформляется в виде кортежа:

$$\Sigma_0 = (M^{\ast}, Q, C^{\ast}, E(m), D(c) \mid V(E(m), D(c)))$$

где:
- $M^{\ast}$ — множество всех открытых текстов (сообщений) длины $k$;
- $Q$ — алфавит $\{0, 1\}$;
- $C^{\ast}$ — множество кодовых слов (шифротекстов) длины $n + 1$;
- $E(m)$ — алгоритм кодирования (шифрования): $m \to c$;
- $D(c)$ — алгоритм декодирования (дешифрования): $c \to m$;
- $V(E(m), D(c))$ — верификация: $D(E(m)) = m$ для любого $m \in M^{\ast}$.

Альтернативным обозначением алгоритмов $E(m)$ и $D(c)$ для алфавитной криптосистемы являются функции $K_F(m|F_0)$ и $K_F(\text{п}|F_0)$ соответственно — как принято считать в классической криптографии, их иначе называют ключами (или функциями) шифрования и дешифрования соответственно.

## 2. Математическое описание

### 2.1. Параметры кода

Расширенный код Хэмминга $C_{[n+1, k]}$ строится на основе стандартного кода $C(n, k)$.

Число проверочных бит $r$ определяется из неравенства:

$$2^r \geq k + r + 1$$

Длина стандартного кодового слова:

$$n = k + r$$

Длина расширенного кодового слова:

$$n_{\text{ext}} = n + 1 = k + r + 1$$

Для классических кодов: $n = 2^r - 1$, $k = 2^r - 1 - r$, $n_{\text{ext}} = 2^r$.

### 2.2. Кодирование $E(m)$

Дано сообщение $m = (m_1, m_2, \ldots, m_k)$, где $m_i \in \{0, 1\}$.

**Шаг 1.** Информационные биты размещаются на позициях кодового слова, **не являющихся** степенями двойки:

$$\text{Позиции информационных бит: } i \in \{1, 2, \ldots, n\} \setminus \{2^0, 2^1, \ldots, 2^{r-1}\}$$

**Шаг 2.** Проверочные биты $p_j$ размещаются на позициях $2^j$ ($j = 0, 1, \ldots, r-1$) и вычисляются как:

$$p_j = \bigoplus_{\substack{i=1 \\ \text{бит } j \text{ в } i = 1}}^{n} c_i, \quad j = 0, 1, \ldots, r-1$$

где $\oplus$ — операция XOR (сложение по модулю 2), а сумма берётся по всем позициям $i$, в двоичном представлении которых $j$-й бит равен 1 (кроме самой позиции $2^j$).

**Шаг 3 (расширение).** Вычисляется общий бит чётности:

$$p_{\text{overall}} = \bigoplus_{i=1}^{n} c_i$$

Расширенное кодовое слово:

$$c_{\text{ext}} = (p_{\text{overall}}, c_1, c_2, \ldots, c_n)$$

### 2.3. Декодирование $D(c)$

Принято расширенное слово $c'_{\text{ext}} = (p'_{\text{overall}}, c'_1, c'_2, \ldots, c'_n)$, возможно, содержащее ошибки.

**Шаг 1.** Вычисляется **синдром** $S = (s_{r-1}, \ldots, s_1, s_0)_2$ стандартного кода:

$$s_j = \bigoplus_{\substack{i=1 \\ \text{бит } j \text{ в } i = 1}}^{n} c'_i, \quad j = 0, 1, \ldots, r-1$$

**Шаг 2.** Вычисляется проверка общей чётности:

$$P = p'_{\text{overall}} \oplus \bigoplus_{i=1}^{n} c'_i$$

**Шаг 3.** Таблица решений:

| Синдром $S$ | Общая чётность $P$ | Интерпретация |
|---|---|---|
| $S = 0$ | $P = 0$ | Ошибок нет |
| $S = 0$ | $P = 1$ | Ошибка в бите $p_{\text{overall}}$ (данные корректны) |
| $S \neq 0$ | $P = 1$ | Одиночная ошибка на позиции $S$ — **исправляем**: $c'_S := c'_S \oplus 1$ |
| $S \neq 0$ | $P = 0$ | Двойная ошибка — **обнаружена**, но не исправляема |

**Шаг 4.** Извлекаем информационные биты из (возможно, исправленного) слова — позиции, не являющиеся степенями 2.

### 2.4. Верификация $V(E(m), D(c))$

Свойство корректности криптосистемы:

$$\forall m \in M^{\ast}: \quad D(E(m)) = m$$

### 2.5. Минимальное расстояние Хэмминга

Стандартный код Хэмминга имеет минимальное расстояние $d_{\min} = 3$, что позволяет исправлять $t = \lfloor(d_{\min} - 1)/2\rfloor = 1$ ошибку.

Расширенный код имеет $d_{\min} = 4$, что позволяет:
- исправлять 1 ошибку: $t = \lfloor(4 - 1)/2\rfloor = 1$;
- обнаруживать до 2 ошибок: $d_{\min} - 1 = 3 \geq 2$.

## 3. Реализация (код)

```python
"""
Расширенный код Хэмминга C[n+1, k] — передача данных по каналу связи
с обнаружением и исправлением одной симметричной ошибки,
а также обнаружением двойной ошибки.

Отличие от стандартного кода Хэмминга C(n, k):
к кодовому слову длины n добавляется один общий бит чётности (позиция 0),
равный XOR всех остальных бит. Итоговая длина — n+1.

Оформлено в соответствии с моделью алфавитной криптосистемы
В.О. Осипяна: Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))).

Автор алгоритма: В.О. Осипян
"""

from __future__ import annotations

from dataclasses import dataclass


# ---------------------------------------------------------------------------
#  Вспомогательные функции
# ---------------------------------------------------------------------------

def _calc_parity_bits(k: int) -> int:
    """Вычисляет минимальное число проверочных бит r для k информационных.

    Условие Хэмминга: 2^r >= k + r + 1.
    Перебираем r, пока неравенство не выполнится.
    """
    # Прямая реализация неравенства 2^r >= k + r + 1.
    # 1 << r — это 2^r в битовой арифметике.
    r = 0
    while (1 << r) < k + r + 1:
        r += 1
    return r


def _is_power_of_two(x: int) -> bool:
    """Проверяет, является ли x степенью двойки.

    У степеней двойки ровно один установленный бит,
    поэтому x & (x - 1) == 0.
    """
    return x > 0 and (x & (x - 1)) == 0


# ---------------------------------------------------------------------------
#  Класс-криптосистема расширенного Хэмминга
# ---------------------------------------------------------------------------

@dataclass
class ExtendedHammingCodeSystem:
    """Реализация криптосистемы Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c)))
    на основе расширенного кода Хэмминга C[n+1, k].

    Атрибуты
    ---------
    k : int
        Число информационных бит (размерность кода).
    r : int
        Число проверочных бит стандартного кода Хэмминга.
    n : int
        Длина стандартного кодового слова (n = k + r).
    n_ext : int
        Длина расширенного кодового слова (n_ext = n + 1 = k + r + 1).

    В терминологии Осипяна
    ----------------------
    M*  — множество всех сообщений (двоичных строк длины k).
    Q   — алфавит {0, 1}.
    C*  — множество кодовых слов (шифротекстов) длины n+1.
    E(m) — алгоритм кодирования (шифрования): m -> c.
    D(c) — алгоритм декодирования (дешифрования): c -> m.
    V(E(m), D(c)) — верификация: D(E(m)) == m для любого m ∈ M*.
    """

    k: int  # число информационных бит
    r: int = 0  # число проверочных бит (вычисляется автоматически)
    n: int = 0  # длина стандартного кодового слова
    n_ext: int = 0  # длина расширенного кодового слова (n + 1)

    def __post_init__(self) -> None:
        if self.k < 1:
            raise ValueError(f"k должно быть >= 1, получено {self.k}")
        self.r = _calc_parity_bits(self.k)
        self.n = self.k + self.r
        self.n_ext = self.n + 1  # +1 за счёт общего бита чётности

    # ---- E(m): кодирование (шифрование) ---------------------------------

    def encode(self, message: list[int]) -> list[int]:
        """Кодирует информационное слово m длины k в расширенное кодовое
        слово c длины n+1.

        Это функция E(m) криптосистемы — прямое преобразование.
        """
        self._validate_message(message)

        # Массив длины n+1 (индексация с 1 для соответствия теории)
        codeword = [0] * (self.n + 1)

        # Шаг 1: информационные биты → позиции, НЕ являющиеся степенями 2
        j = 0
        for i in range(1, self.n + 1):
            if not _is_power_of_two(i):
                codeword[i] = message[j]
                j += 1

        # Шаг 2: проверочные биты p_j на позициях 2^j
        # p_j = XOR всех codeword[i], где j-й бит числа i = 1
        for j in range(self.r):
            parity_pos = 1 << j
            parity = 0
            for i in range(1, self.n + 1):
                if i & parity_pos and i != parity_pos:
                    parity ^= codeword[i]
            codeword[parity_pos] = parity

        # Шаг 3 (расширение): общий бит чётности = XOR всех n бит
        # Именно этот бит отличает расширенный код от стандартного
        # и обеспечивает обнаружение двойных ошибок.
        p_overall = 0
        for i in range(1, self.n + 1):
            p_overall ^= codeword[i]

        # Формат: [p_overall, c_1, c_2, ..., c_n]
        return [p_overall] + codeword[1:]

    # ---- D(c): декодирование (дешифрование) ------------------------------

    def decode(self, received: list[int]) -> tuple[list[int], int, str]:
        """Декодирует принятое расширенное слово длины n+1.

        Это функция D(c) криптосистемы — обратное преобразование.

        Возвращает (decoded_message, error_position, status):
        - status: "no_error" / "corrected" / "double_error"
        - error_position: 0 = нет ошибки, >0 = позиция, -1 = двойная
        """
        self._validate_extended_codeword(received)

        # Разделяем p_overall и основное кодовое слово
        p_overall_received = received[0]
        cw = [0] + list(received[1:])

        # Шаг 1: синдром S стандартного кода
        syndrome = 0
        for j in range(self.r):
            parity_pos = 1 << j
            s_j = 0
            for i in range(1, self.n + 1):
                if i & parity_pos:
                    s_j ^= cw[i]
            if s_j:
                syndrome |= parity_pos

        # Шаг 2: проверка общей чётности
        # P = p_overall ⊕ XOR(c_1, ..., c_n)
        p_check = p_overall_received
        for i in range(1, self.n + 1):
            p_check ^= cw[i]

        # Шаг 3: таблица решений (ключевое отличие расширенного кода)
        # S=0, P=0 → OK | S=0, P=1 → ошибка в p_overall
        # S≠0, P=1 → 1 ошибка | S≠0, P=0 → 2 ошибки
        if syndrome == 0 and p_check == 0:
            error_position = 0
            status = "no_error"
        elif syndrome == 0 and p_check == 1:
            error_position = 0
            status = "corrected"
        elif syndrome != 0 and p_check == 1:
            if syndrome > self.n:
                raise ValueError(
                    f"Синдром {syndrome} выходит за пределы "
                    f"кодового слова длины {self.n}. "
                    "Возможно, произошло более двух ошибок."
                )
            cw[syndrome] ^= 1
            error_position = syndrome
            status = "corrected"
        else:  # syndrome != 0, p_check == 0
            error_position = -1
            status = "double_error"

        # Шаг 4: извлекаем информационные биты
        decoded = []
        for i in range(1, self.n + 1):
            if not _is_power_of_two(i):
                decoded.append(cw[i])

        return decoded, error_position, status

    # ---- V(E(m), D(c)): верификация --------------------------------------

    def verify(self, message: list[int]) -> bool:
        """Проверяет D(E(m)) == m для данного m."""
        self._validate_message(message)
        encoded = self.encode(message)
        decoded, _, status = self.decode(encoded)
        return decoded == message and status == "no_error"

    # ---- Валидация -------------------------------------------------------

    def _validate_message(self, message: list[int]) -> None:
        if len(message) != self.k:
            raise ValueError(
                f"Длина сообщения должна быть {self.k}, получено {len(message)}"
            )
        if not all(b in (0, 1) for b in message):
            raise ValueError("Сообщение должно содержать только 0 и 1")

    def _validate_extended_codeword(self, codeword: list[int]) -> None:
        if len(codeword) != self.n_ext:
            raise ValueError(
                f"Длина расширенного кодового слова должна быть {self.n_ext}, "
                f"получено {len(codeword)}"
            )
        if not all(b in (0, 1) for b in codeword):
            raise ValueError("Кодовое слово должно содержать только 0 и 1")

    def __repr__(self) -> str:
        return (
            f"ExtendedHammingCodeSystem(C[{self.n_ext},{self.k}], "
            f"n={self.n}, r={self.r}, "
            f"исправляет 1, обнаруживает 2 ошибки)"
        )
```

## 4. Связь кода с теорией

| Математическая формула | Строка кода | Пояснение |
|---|---|---|
| $2^r \geq k + r + 1$ | `while (1 << r) < k + r + 1` | Вычисление минимального $r$. `1 << r` — это $2^r$ в битовой арифметике. |
| $n_{\text{ext}} = n + 1$ | `self.n_ext = self.n + 1` | Длина расширенного кодового слова на 1 больше стандартного — за счёт бита общей чётности. |
| Позиции проверочных бит: $2^0, 2^1, \ldots, 2^{r-1}$ | `_is_power_of_two(i)` | Отделяем позиции проверочных бит от информационных. Степень двойки: `x & (x-1) == 0`. |
| $p_j = \bigoplus c_i$ (бит $j$ в $i = 1$) | `parity ^= codeword[i]` внутри `if i & parity_pos` | Вычисление проверочного бита. Условие `i & parity_pos` проверяет $j$-й бит позиции $i$. `^=` — XOR ($\oplus$). |
| $p_{\text{overall}} = \bigoplus_{i=1}^{n} c_i$ | `p_overall ^= codeword[i]` | **Ключевое отличие** расширенного кода: общий бит чётности — XOR всех $n$ бит стандартного слова. |
| Синдром $s_j = \bigoplus c'_i$ | `s_j ^= cw[i]` | Вычисление $j$-го бита синдрома. Включаем все позиции (и проверочные). |
| $P = p'_{\text{overall}} \oplus \bigoplus c'_i$ | `p_check ^= cw[i]` (начиная с `p_check = p_overall_received`) | Проверка общей чётности принятого слова. |
| Таблица решений ($S$, $P$) | Блок `if/elif/else` | Четыре случая: нет ошибки, ошибка в $p_{\text{overall}}$, одиночная ошибка (исправляем), двойная ошибка (обнаруживаем). |
| $c'_S := c'_S \oplus 1$ | `cw[syndrome] ^= 1` | Инвертируем бит на позиции синдрома для исправления одиночной ошибки. |
| $D(E(m)) = m$ | `decoded == message` в `verify()` | Прямая проверка свойства верификации $V(E(m), D(c))$. |

## 5. Пример работы

### Расширенный код Хэмминга $C_{[8, 4]}$: $k = 4$, $r = 3$, $n = 7$, $n_{\text{ext}} = 8$

**Кодирование** сообщения $m = (1, 0, 1, 1)$:

Позиции стандартного кодового слова (нумерация с 1):

| Позиция | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|
| Тип | $p_0$ | $p_1$ | $m_1$ | $p_2$ | $m_2$ | $m_3$ | $m_4$ |
| Значение | 0 | 1 | **1** | 0 | **0** | **1** | **1** |

Общий бит чётности: $p_{\text{overall}} = 0 \oplus 1 \oplus 1 \oplus 0 \oplus 0 \oplus 1 \oplus 1 = 0$

Расширенное кодовое слово: $c = E(m) = (0, 0, 1, 1, 0, 0, 1, 1)$

**Декодирование с одиночной ошибкой** на позиции 5:

Принято: $c' = (0, 0, 1, 1, 0, \mathbf{1}, 1, 1)$

- Синдром: $S = (1, 0, 1)_2 = 5$
- Общая чётность: $P = 1$ (нечётная)
- $S \neq 0$ и $P = 1$ → одиночная ошибка на позиции 5 → **исправляем**
- Результат: $m = (1, 0, 1, 1)$ ✓

**Декодирование с двойной ошибкой** на позициях 2 и 5:

Принято: $c' = (0, 0, \mathbf{0}, 1, 0, \mathbf{1}, 1, 1)$

- Синдром: $S = (1, 1, 1)_2 = 7 \neq 0$
- Общая чётность: $P = 0$ (чётная)
- $S \neq 0$ и $P = 0$ → **двойная ошибка обнаружена** (не исправляема)
