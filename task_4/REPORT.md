# Алгоритм RSA — асимметричная криптосистема с открытым ключом

## 1. Идея метода (схематично)

Алгоритм RSA (Rivest–Shamir–Adleman) — это асимметричная криптосистема с открытым ключом, в которой для **шифрования** и **дешифрования** используются **разные ключи**: открытый (публичный) и закрытый (секретный).

### Ключевая идея

Безопасность RSA основана на **NP-сложной задаче факторизации** больших целых чисел: имея произведение $n = p \cdot q$ двух больших простых чисел, вычислительно невозможно за полиномиальное время найти $p$ и $q$.

### Схема работы криптосистемы

```
Отправитель (Алиса)                                    Получатель (Боб)
┌───────────────────┐                              ┌───────────────────┐
│ Генерация ключей  │                              │                   │
│  1. Выбрать p, q  │                              │                   │
│     (простые)     │                              │                   │
│  2. n = p·q       │     Открытый ключ (e, n)     │                   │
│  3. φ(n)=(p-1)(q-1)│ ─────────────────────────► │                   │
│  4. Выбрать e:    │                              │                   │
│     gcd(e,φ(n))=1 │                              │                   │
│  5. d = e⁻¹ mod φ │                              │                   │
└───────────────────┘                              └───────────────────┘
        │                                                   │
        │ Закрытый ключ (d, n)                              │
        │ хранится в секрете                                │
        │                                                   │
        ▼                                                   ▼
┌───────────────────┐     Шифротекст c            ┌───────────────────┐
│  Шифрование E(m)  │ ──────────────────────────► │  Дешифрование D(c)│
│  c = m^e mod n    │     (открытый канал)         │  m = c^d mod n    │
└───────────────────┘                              └───────────────────┘
```

В терминологии криптосистемы Осипяна задача оформляется в виде кортежа:

$$\Sigma_0 = (M^{\ast}, Q, C^{\ast}, E(m), D(c) \mid V(E(m), D(c)))$$

где:
- $M^{\ast}$ — множество всех открытых текстов (целых чисел от $0$ до $n-1$);
- $Q$ — алфавит $\{0, 1, \ldots, n-1\}$ (числовые эквиваленты элементарных сообщений);
- $C^{\ast}$ — множество шифротекстов (целых чисел от $0$ до $n-1$);
- $E(m)$ — алгоритм шифрования: $c = m^e \bmod n$;
- $D(c)$ — алгоритм дешифрования: $m = c^d \bmod n$;
- $V(E(m), D(c))$ — верификация: $D(E(m)) = m$ для любого $m \in M^{\ast}$.

Алгоритмы $E(m)$ и $D(c)$ связаны тем, что для любого сообщения $m = m_1 m_2 \ldots m_k \in M^{\ast}$ однозначно преобразуется в шифротекст $c = c_1 c_2 \ldots c_k \in C^{\ast}$, и обратно: по шифротексту $c$ всегда можно восстановить исходное сообщение $m$.

Альтернативным обозначением алгоритмов $E(m)$ и $D(c)$ для алфавитной криптосистемы являются функции $K_F$ (шифрование, «ключевая функция») и $K_F^{-1}$ (дешифрование, «обратная ключевая функция»), что соответствует понятиям ключей шифрования и дешифрования в классической криптографии.

## 2. Математическое описание

### 2.1. Генерация ключей

**Шаг 1.** Выбираются два различных больших простых числа $p$ и $q$.

**Шаг 2.** Вычисляется модуль RSA:

$$n = p \cdot q$$

**Шаг 3.** Вычисляется функция Эйлера:

$$\varphi(n) = (p - 1)(q - 1)$$

Функция Эйлера $\varphi(n)$ показывает количество натуральных чисел, меньших $n$ и взаимно простых с $n$. Для $n = p \cdot q$ (произведение двух различных простых) формула упрощается до $(p-1)(q-1)$.

**Шаг 4.** Выбирается открытая экспонента $e$:

$$1 < e < \varphi(n), \quad \gcd(e, \varphi(n)) = 1$$

Стандартное значение $e = 65537 = 2^{16} + 1$ (четвёртое простое число Ферма $F_4$).

**Шаг 5.** Вычисляется секретная экспонента $d$ — мультипликативный обратный элемент $e$ по модулю $\varphi(n)$:

$$d = e^{-1} \bmod \varphi(n), \quad \text{т.е.} \quad e \cdot d \equiv 1 \pmod{\varphi(n)}$$

Нахождение $d$ выполняется с помощью **расширенного алгоритма Евклида**.

### 2.2. Шифрование $E(m)$

Для открытого текста $m \in \{0, 1, \ldots, n-1\}$:

$$c = E(m) = m^e \bmod n$$

### 2.3. Дешифрование $D(c)$

Для шифротекста $c \in \{0, 1, \ldots, n-1\}$:

$$m = D(c) = c^d \bmod n$$

### 2.4. Доказательство корректности $V(E(m), D(c))$

Необходимо показать, что $D(E(m)) = m$ для всех $m \in M^{\ast}$.

$$D(E(m)) = (m^e)^d \bmod n = m^{e \cdot d} \bmod n$$

Поскольку $e \cdot d \equiv 1 \pmod{\varphi(n)}$, существует целое $k$ такое, что $e \cdot d = 1 + k \cdot \varphi(n)$.

Тогда по **теореме Эйлера** (если $\gcd(m, n) = 1$):

$$m^{\varphi(n)} \equiv 1 \pmod{n}$$

Следовательно:

$$m^{e \cdot d} = m^{1 + k\varphi(n)} = m \cdot (m^{\varphi(n)})^k \equiv m \cdot 1^k = m \pmod{n}$$

### 2.5. NP-сложная задача факторизации

Криптостойкость RSA основана на вычислительной сложности задачи **целочисленной факторизации** (Integer Factorization Problem, IFP):

> **Задача IFP:** Дано составное число $n$. Найти его нетривиальные делители $p$ и $q$ такие, что $n = p \cdot q$.

Эта задача принадлежит к классу **NP**: если кто-то сообщает нам $p$ и $q$, мы можем быстро (за полиномиальное время) проверить, что $p \cdot q = n$. Однако **не известен полиномиальный алгоритм** для её решения на классическом компьютере.

#### Сложность наилучших алгоритмов

Наилучший известный классический алгоритм — **решето числового поля** (General Number Field Sieve, GNFS) — имеет **субэкспоненциальную** сложность:

$$T(n) = \exp\left(c \cdot (\ln n)^{1/3} \cdot (\ln \ln n)^{2/3}\right)$$

где $c \approx 1.923$ для факторизации.

Это означает:
- Для 512-битного $n$ факторизация возможна (RSA-155 факторизовано в 1999 г.).
- Для 1024-битного $n$ факторизация находится на грани возможного.
- Для 2048-битного $n$ и выше — вычислительно неосуществима при текущих технологиях.

#### Связь факторизации и взлома RSA

Если злоумышленник факторизует $n = p \cdot q$, он может:
1. Вычислить $\varphi(n) = (p-1)(q-1)$.
2. Вычислить $d = e^{-1} \bmod \varphi(n)$ (расширенный алгоритм Евклида).
3. Расшифровать любое сообщение: $m = c^d \bmod n$.

Таким образом, **взлом RSA сводится к задаче факторизации**, а задача факторизации является NP-сложной.

## 3. Реализация (код)

```python
"""
Алгоритм RSA — асимметричная криптосистема с открытым ключом.

Оформлено в соответствии с моделью алфавитной криптосистемы
В.О. Осипяна: Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))).

NP-сложная задача в основе RSA — задача факторизации целых чисел:
разложение большого числа n = p·q на простые множители.

Литература: Осипян В.О. Разработка методов построения систем передачи
            и защиты информации. Монография. КубГУ, 2004.
"""

from __future__ import annotations

import math
import random
import sys
from dataclasses import dataclass, field


# ---------------------------------------------------------------------------
#  Вспомогательные функции теории чисел
# ---------------------------------------------------------------------------

def is_prime(n: int) -> bool:
    """Проверяет, является ли число простым (детерминистический тест).

    Для чисел до ~10^18 достаточно перебора делителей до √n.
    Для учебных целей (ключи до нескольких сотен бит) этого достаточно.
    """
    if n < 2:
        return False
    if n < 4:
        return True
    if n % 2 == 0 or n % 3 == 0:
        return False
    # Проверяем делители вида 6k±1 до √n
    i = 5
    while i * i <= n:
        if n % i == 0 or n % (i + 2) == 0:
            return False
        i += 6
    return True


def generate_prime(bits: int) -> int:
    """Генерирует случайное простое число заданной битовой длины.

    Используется вероятностный подход: генерируем нечётные числа
    нужной длины и проверяем на простоту.
    """
    if bits < 2:
        raise ValueError("Битовая длина должна быть >= 2")
    while True:
        # Старший бит = 1 (гарантируем нужную длину), младший = 1 (нечётное)
        n = random.getrandbits(bits)
        n |= (1 << (bits - 1)) | 1
        if is_prime(n):
            return n


def gcd(a: int, b: int) -> int:
    """Наибольший общий делитель (алгоритм Евклида)."""
    while b:
        a, b = b, a % b
    return a


def extended_gcd(a: int, b: int) -> tuple[int, int, int]:
    """Расширенный алгоритм Евклида.

    Возвращает (g, x, y) такие, что a·x + b·y = g = gcd(a, b).

    Это ключевой алгоритм для нахождения секретной экспоненты d,
    поскольку d = e⁻¹ mod φ(n), т.е. нужно решить e·d ≡ 1 (mod φ(n)).
    """
    if a == 0:
        return b, 0, 1
    g, x, y = extended_gcd(b % a, a)
    return g, y - (b // a) * x, x


def mod_inverse(e: int, phi: int) -> int:
    """Вычисляет мультипликативный обратный элемент: d = e⁻¹ mod φ.

    Использует расширенный алгоритм Евклида.
    Если обратный не существует (gcd(e, φ) ≠ 1), выбрасывает исключение.
    """
    g, x, _ = extended_gcd(e % phi, phi)
    if g != 1:
        raise ValueError(
            f"Обратный элемент не существует: gcd({e}, {phi}) = {g} ≠ 1"
        )
    return x % phi


def mod_pow(base: int, exp: int, mod: int) -> int:
    """Быстрое модульное возведение в степень (метод двоичного разложения).

    Вычисляет base^exp mod mod за O(log exp) умножений.
    Это основная операция шифрования и дешифрования RSA:
      c = m^e mod n  (шифрование)
      m = c^d mod n  (дешифрование)
    """
    # Используем встроенную функцию Python pow(base, exp, mod),
    # которая реализует тот же алгоритм, но оптимизирована на уровне C
    return pow(base, exp, mod)


# ---------------------------------------------------------------------------
#  Класс-криптосистема RSA
# ---------------------------------------------------------------------------

@dataclass
class RSACryptoSystem:
    """Реализация криптосистемы Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c)))
    на основе алгоритма RSA.

    Алгоритм RSA основан на NP-сложной задаче факторизации:
    зная n = p·q, вычислительно невозможно найти p и q за полиномиальное
    время при достаточно больших простых числах.

    В терминологии Осипяна
    ----------------------
    M*   — множество всех открытых текстов (целых чисел от 0 до n−1).
    Q    — алфавит {0, 1, ..., n−1} (числовые эквиваленты сообщений).
    C*   — множество шифротекстов (целых чисел от 0 до n−1).
    E(m) — алгоритм шифрования: c = m^e mod n.
    D(c) — алгоритм дешифрования: m = c^d mod n.
    V(E(m), D(c)) — верификация: D(E(m)) = m для любого m ∈ M*.
    """

    p: int
    q: int
    e: int = 0

    def __post_init__(self) -> None:
        """Генерация ключей RSA из заданных простых чисел p и q."""
        # Валидация: p и q должны быть различными простыми числами
        if not is_prime(self.p):
            raise ValueError(f"p = {self.p} не является простым числом")
        if not is_prime(self.q):
            raise ValueError(f"q = {self.q} не является простым числом")
        if self.p == self.q:
            raise ValueError("p и q должны быть различными простыми числами")

        # Шаг 1: n = p·q — модуль RSA
        self.n = self.p * self.q

        # Шаг 2: φ(n) = (p−1)(q−1) — функция Эйлера
        self.phi = (self.p - 1) * (self.q - 1)

        # Шаг 3: выбор открытой экспоненты e
        if self.e == 0:
            self.e = self._choose_e()
        else:
            if gcd(self.e, self.phi) != 1:
                raise ValueError(
                    f"e = {self.e} не взаимно просто с φ(n) = {self.phi}"
                )

        # Шаг 4: d = e⁻¹ mod φ(n) — секретная экспонента
        self.d = mod_inverse(self.e, self.phi)

    # ---- E(m): шифрование ----
    def encrypt(self, m: int) -> int:
        """c = m^e mod n  —  функция E(m) криптосистемы."""
        return mod_pow(m, self.e, self.n)

    # ---- D(c): дешифрование ----
    def decrypt(self, c: int) -> int:
        """m = c^d mod n  —  функция D(c) криптосистемы."""
        return mod_pow(c, self.d, self.n)

    # ---- V(E(m), D(c)): верификация ----
    def verify(self, m: int) -> bool:
        """Проверяет D(E(m)) == m."""
        c = self.encrypt(m)
        return self.decrypt(c) == m
```

## 4. Связь кода с теорией

### 4.1. Генерация ключей

| Математическая операция | Строка кода | Пояснение |
|---|---|---|
| $n = p \cdot q$ | `self.n = self.p * self.q` | Модуль RSA — произведение двух простых |
| $\varphi(n) = (p-1)(q-1)$ | `self.phi = (self.p - 1) * (self.q - 1)` | Функция Эйлера |
| $\gcd(e, \varphi(n)) = 1$ | `gcd(self.e, self.phi) != 1` | Проверка взаимной простоты |
| $d = e^{-1} \bmod \varphi(n)$ | `self.d = mod_inverse(self.e, self.phi)` | Расширенный алгоритм Евклида |

### 4.2. Расширенный алгоритм Евклида

Для нахождения $d = e^{-1} \bmod \varphi(n)$ необходимо решить уравнение:

$$e \cdot d + \varphi(n) \cdot y = 1$$

Это делается рекурсивной функцией `extended_gcd(a, b)`, которая возвращает $(g, x, y)$ такие, что $a \cdot x + b \cdot y = g$.

```python
def extended_gcd(a: int, b: int) -> tuple[int, int, int]:
    if a == 0:
        return b, 0, 1          # База: 0·x + b·1 = b
    g, x, y = extended_gcd(b % a, a)
    return g, y - (b // a) * x, x  # Обратный ход: пересчёт коэффициентов
```

Затем `mod_inverse` берёт $x \bmod \varphi(n)$, чтобы получить положительное $d$:

```python
def mod_inverse(e: int, phi: int) -> int:
    g, x, _ = extended_gcd(e % phi, phi)
    return x % phi   # d = x mod φ(n)
```

### 4.3. Шифрование и дешифрование

| Формула | Код | Операция |
|---|---|---|
| $c = m^e \bmod n$ | `pow(m, self.e, self.n)` | Шифрование E(m) |
| $m = c^d \bmod n$ | `pow(c, self.d, self.n)` | Дешифрование D(c) |

Функция `pow(base, exp, mod)` в Python реализует **быстрое модульное возведение в степень** методом двоичного разложения показателя, что обеспечивает сложность $O(\log e)$ умножений.

### 4.4. Верификация $V(E(m), D(c))$

Верификация основана на теореме Эйлера. В коде:

```python
def verify(self, m: int) -> bool:
    c = self.encrypt(m)       # c = m^e mod n
    return self.decrypt(c) == m  # m ?= c^d mod n = m^(e·d) mod n
```

Корректность: $m^{e \cdot d} = m^{1 + k\varphi(n)} = m \cdot (m^{\varphi(n)})^k \equiv m \pmod{n}$.

### 4.5. Посимвольное шифрование текста

Каждый символ текста $m_i$ кодируется как его Unicode-номер (числовой эквивалент элементарного сообщения), после чего шифруется индивидуально:

$$c_i = E(m_i) = m_i^e \bmod n$$

В коде:

```python
def encrypt_text(self, text: str) -> list[int]:
    result = []
    for char in text:
        code = ord(char)       # m_i — числовой эквивалент символа
        result.append(self.encrypt(code))  # c_i = m_i^e mod n
    return result
```

Дешифрование:

$$m_i = D(c_i) = c_i^d \bmod n, \quad \text{символ} = \text{chr}(m_i)$$

### 4.6. Факторизация и криптостойкость

Функция `trial_factorize` демонстрирует простейший метод факторизации — **пробное деление** со сложностью $O(\sqrt{n})$:

```python
def trial_factorize(n: int, limit: int = 10**6) -> tuple[int, int] | None:
    for i in range(3, min(limit, int(n**0.5) + 1), 2):
        if n % i == 0:
            return (i, n // i)
    return None
```

Для больших $n$ (≥ 2048 бит) этот метод неэффективен. Наилучший известный алгоритм GNFS имеет субэкспоненциальную сложность $T(n) = \exp(c \cdot (\ln n)^{1/3} \cdot (\ln \ln n)^{2/3})$, что делает факторизацию вычислительно неосуществимой.

## 5. Криптостойкость алгоритма RSA

### 5.1. Основа криптостойкости

Криптостойкость RSA базируется на **вычислительной односторонности** задачи факторизации:
- **Прямая операция** (умножение $p \cdot q$) выполняется за $O(\log^2 n)$ — полиномиальное время.
- **Обратная операция** (факторизация $n$ на $p$ и $q$) не имеет известного полиномиального алгоритма.

### 5.2. Рекомендуемые размеры ключей

| Размер ключа $n$ (бит) | Уровень безопасности | Статус |
|---|---|---|
| 512 | ~30 бит | **Взломан** (RSA-155, 1999) |
| 1024 | ~80 бит | **Не рекомендуется** |
| 2048 | ~112 бит | Безопасен до ~2030 г. |
| 4096 | ~140 бит | Высокий уровень безопасности |

### 5.3. Известные атаки

1. **Факторизация** — основная атака, сводящаяся к NP-сложной задаче.
2. **Атака Винера** — при малом $d$ ($d < n^{0.25} / 3$).
3. **Атака общим модулем** — при использовании одного $n$ для разных пользователей.
4. **Квантовый алгоритм Шора** — теоретически факторизует за полиномиальное время на квантовом компьютере.
