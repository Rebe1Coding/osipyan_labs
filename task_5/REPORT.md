# Алгоритм классического аддитивного рюкзака (АВКР) — криптосистема Меркла-Хеллмана

## Идея метода (схематично)

Криптосистема Меркла-Хеллмана основана на задаче о рюкзаке (Subset Sum Problem). Идея состоит в том, чтобы преобразовать **«лёгкую»** задачу (рюкзак для сверхвозрастающей последовательности) в **«трудную»** (рюкзак для произвольной последовательности) с помощью модулярной арифметики.

```
Генерация ключей:
  B (сверхвозрастающая) ──[× w mod q]──> A (открытый ключ)
       закрытый ключ                     выглядит как случайная

Шифрование (отправитель знает только A):
  m = (m₁, m₂, ..., mₙ) ──[c = Σ mᵢ·aᵢ]──> c (шифротекст)

Дешифрование (получатель знает B, q, w):
  c ──[S' = c·w⁻¹ mod q]──> S' ──[жадный алгоритм по B]──> m
```

**Почему это работает:** сверхвозрастающая последовательность — это частный случай, для которого задача о рюкзаке решается жадным алгоритмом за O(n). Умножение на `w mod q` «маскирует» структуру, превращая задачу в NP-трудную для стороннего наблюдателя. Обладатель секрета (w, q) может «снять маску» и решить лёгкий рюкзак.

## NP-сложная задача в основе АВКР

В основе АВКР лежит **задача о сумме подмножеств** (Subset Sum Problem, SSP):

> Дан набор положительных целых чисел $\{a_1, a_2, \ldots, a_n\}$ и целое число $S$. Требуется определить, существует ли подмножество $I \subseteq \{1, 2, \ldots, n\}$, такое что $\sum_{i \in I} a_i = S$.

Эта задача является **NP-полной** (доказано Р. Карпом в 1972 г.) — одной из 21 задачи в оригинальном списке Карпа. Для произвольного набора чисел наилучшие известные алгоритмы:

- **Полный перебор:** $O(2^n)$
- **Meet-in-the-middle (Хоровиц-Сахни):** $O(2^{n/2})$
- **Динамическое программирование:** $O(n \cdot S)$ (псевдополиномиальный)

## Криптостойкость АВКР

Криптостойкость АВКР основана на предположении, что решение задачи о рюкзаке для **общей** (не сверхвозрастающей) последовательности вычислительно неосуществимо за полиномиальное время.

**Важное замечание:** в 1982 г. А. Шамир (Shamir) показал, что оригинальная схема Меркла-Хеллмана может быть взломана за полиномиальное время с использованием **LLL-алгоритма** (алгоритма Ленстра-Ленстра-Ловаса) для приведения базиса решёток. Это означает, что одноитерационная схема АВКР **не является криптографически стойкой** по современным стандартам.

Тем не менее задача о рюкзаке остаётся NP-полной в общем случае, и существуют модифицированные рюкзачные криптосистемы (многоитерационные, на основе решёток), которые считаются более стойкими.

## Математическое описание

### Модель алфавитной криптосистемы В.О. Осипяна

Криптосистема оформлена в виде кортежа:

$$\Sigma_D = (M^{\ast}, Q, C^{\ast}, E(m), D(c) \mid V(E(m), D(c)))$$

где:

| Обозначение | Описание |
|---|---|
| $M^{\ast}$ | Множество всех открытых текстов $m = (m_1, m_2, \ldots, m_n)$, $m_i \in \{0, 1\}$ |
| $Q$ | Алфавит $\{0, 1\}$ — числовые эквиваленты элементов сообщений |
| $C^{\ast}$ | Множество шифротекстов — целых чисел $c = \sum m_i \cdot a_i$ |
| $E(m)$ | Алгоритм шифрования (прямое преобразование) |
| $D(c)$ | Алгоритм дешифрования (обратное преобразование) |
| $V(E(m), D(c))$ | Верификация: $D(E(m)) = m$ для любого $m \in M^{\ast}$ |

Альтернативные обозначения: $E(m) = K_F(\text{inp } F_0)$, $D(c) = K_F(\text{ran } F_0)$ — ключевые функции шифрования и дешифрования соответственно.

### Генерация ключей

**Шаг 1.** Выбирается **сверхвозрастающая последовательность** $B = (b_1, b_2, \ldots, b_n)$:

$$b_i > \sum_{j=1}^{i-1} b_j, \quad i = 2, 3, \ldots, n$$

**Шаг 2.** Выбирается модуль $q$ — простое число, большее суммы всех элементов $B$:

$$q > \sum_{i=1}^{n} b_i$$

**Шаг 3.** Выбирается множитель $w$, взаимно простой с $q$:

$$\gcd(w, q) = 1, \quad 2 \leq w < q$$

**Шаг 4.** Формируется **открытый ключ** $A = (a_1, a_2, \ldots, a_n)$:

$$a_i = w \cdot b_i \mod q, \quad i = 1, 2, \ldots, n$$

**Закрытый ключ:** $(B, q, w)$. **Открытый ключ:** $A$.

### Шифрование $E(m)$

Для сообщения $m = (m_1, m_2, \ldots, m_n)$, $m_i \in \{0, 1\}$:

$$c = E(m) = \sum_{i=1}^{n} m_i \cdot a_i$$

### Дешифрование $D(c)$

**Шаг 1.** Вычисляем:

$$S' = c \cdot w^{-1} \mod q$$

где $w^{-1}$ — мультипликативный обратный элемент, найденный расширенным алгоритмом Евклида:

$$w \cdot w^{-1} \equiv 1 \pmod{q}$$

Корректность: $S' = c \cdot w^{-1} = \left(\sum m_i \cdot w \cdot b_i\right) \cdot w^{-1} = \sum m_i \cdot b_i \pmod{q}$

Поскольку $q > \sum b_i \geq \sum m_i \cdot b_i$, результат берётся без редукции по модулю.

**Шаг 2.** Решаем **лёгкий рюкзак** для $S'$ и $B$ жадным алгоритмом:

```
для i от n до 1:
    если b_i ≤ S':
        m_i = 1, S' = S' - b_i
    иначе:
        m_i = 0
```

### Верификация $V(E(m), D(c))$

$$D(E(m)) = D\left(\sum m_i \cdot a_i\right) = m$$

для любого $m \in M^{\ast}$, что следует из:

$$\sum m_i \cdot a_i \cdot w^{-1} \equiv \sum m_i \cdot b_i \pmod{q}$$

## Реализация (код)

```python
"""
Алгоритм классического аддитивного рюкзака (АВКР) — криптосистема Меркла-Хеллмана.

Оформлено в соответствии с моделью алфавитной криптосистемы
В.О. Осипяна: Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))).
"""

from __future__ import annotations

import random
import sys
from dataclasses import dataclass, field

from PyQt6.QtWidgets import (
    QApplication, QMainWindow, QWidget, QVBoxLayout, QHBoxLayout,
    QLabel, QLineEdit, QPushButton, QTextEdit, QGroupBox, QSpinBox,
    QMessageBox, QTabWidget,
)
from PyQt6.QtCore import Qt
from PyQt6.QtGui import QFont


# ---------------------------------------------------------------------------
#  Вспомогательные функции теории чисел
# ---------------------------------------------------------------------------

def gcd(a: int, b: int) -> int:
    """Наибольший общий делитель (алгоритм Евклида).

    Используется для проверки взаимной простоты w и q:
    gcd(w, q) = 1 — необходимое условие существования w⁻¹ mod q.
    """
    while b:
        a, b = b, a % b
    return a


def extended_gcd(a: int, b: int) -> tuple[int, int, int]:
    """Расширенный алгоритм Евклида: a·x + b·y = gcd(a, b).

    Ключевой алгоритм для дешифрования: через него находится w⁻¹ mod q,
    необходимый для перехода от «трудного» рюкзака к «лёгкому».
    """
    if a == 0:
        return b, 0, 1
    g, x, y = extended_gcd(b % a, a)
    return g, y - (b // a) * x, x


def mod_inverse(a: int, m: int) -> int:
    """Мультипликативный обратный: a⁻¹ mod m.

    Вычисляет w⁻¹ mod q, используя расширенный алгоритм Евклида.
    Корректность: если gcd(a, m) = 1, то существует x: a·x ≡ 1 (mod m).
    """
    g, x, _ = extended_gcd(a % m, m)
    if g != 1:
        raise ValueError(f"Обратный элемент не существует: gcd({a}, {m}) = {g} ≠ 1")
    return x % m


# ---------------------------------------------------------------------------
#  Генерация сверхвозрастающей последовательности
# ---------------------------------------------------------------------------

def generate_superincreasing_sequence(length: int, start_range: tuple[int, int] = (2, 10)) -> list[int]:
    """Генерирует сверхвозрастающую последовательность B = (b₁, ..., bₙ).

    Свойство: bᵢ > Σⱼ₌₁ⁱ⁻¹ bⱼ  — именно это делает задачу о рюкзаке
    «лёгкой» для данной последовательности (решается жадным алгоритмом).
    Последовательность является закрытым ключом криптосистемы.
    """
    seq = [random.randint(*start_range)]
    total = seq[0]
    for _ in range(1, length):
        # Добавляем случайный отступ, чтобы последовательность была
        # не минимальной сверхвозрастающей (это усложнило бы анализ)
        next_val = total + random.randint(1, max(total, 10))
        seq.append(next_val)
        total += next_val
    return seq


# ---------------------------------------------------------------------------
#  Решение «лёгкого» рюкзака
# ---------------------------------------------------------------------------

def solve_superincreasing_knapsack(target: int, sequence: list[int]) -> list[int] | None:
    """Жадный алгоритм для сверхвозрастающей последовательности.

    Проходим справа налево: если bᵢ ≤ остатка, включаем элемент (mᵢ = 1).
    Сверхвозрастаемость гарантирует единственность и корректность решения.
    Это O(n) — в отличие от O(2ⁿ) для общей задачи.
    """
    n = len(sequence)
    bits = [0] * n
    remainder = target
    for i in range(n - 1, -1, -1):
        if sequence[i] <= remainder:
            bits[i] = 1
            remainder -= sequence[i]
    return bits if remainder == 0 else None


# ---------------------------------------------------------------------------
#  Криптосистема АВКР
# ---------------------------------------------------------------------------

@dataclass
class KnapsackCryptoSystem:
    """Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))) на основе АВКР.

    Закрытый ключ: B (сверхвозрастающая), q (модуль), w (множитель).
    Открытый ключ: A, где aᵢ = w · bᵢ mod q.

    E(m) = Σ mᵢ · aᵢ          — сумма по открытому ключу
    D(c) = solve(c·w⁻¹ mod q, B) — переход к лёгкому рюкзаку
    """
    block_size: int
    private_seq: list[int] = field(default_factory=list)
    q: int = 0
    w: int = 0
    public_seq: list[int] = field(init=False, default_factory=list)
    w_inv: int = field(init=False, default=0)

    def __post_init__(self) -> None:
        # Генерация/валидация закрытой последовательности B
        if not self.private_seq:
            self.private_seq = generate_superincreasing_sequence(self.block_size)
        total = sum(self.private_seq)

        # q > Σ bᵢ — чтобы mod q не «обрезал» сумму при дешифровании
        if self.q == 0:
            self.q = generate_prime_in_range(total + 1, total * 2 + 100)

        # w: gcd(w, q) = 1 — чтобы существовал обратный w⁻¹
        if self.w == 0:
            self.w = random.randint(2, self.q - 1)
            while gcd(self.w, self.q) != 1:
                self.w = random.randint(2, self.q - 1)

        # w⁻¹ mod q — ключевой элемент дешифрования
        self.w_inv = mod_inverse(self.w, self.q)

        # Открытый ключ: aᵢ = w · bᵢ mod q
        # Модулярное умножение «маскирует» сверхвозрастаемость
        self.public_seq = [(self.w * b) % self.q for b in self.private_seq]

    def encrypt_block(self, bits: list[int]) -> int:
        """E(m) = Σᵢ₌₁ⁿ mᵢ · aᵢ — скалярное произведение бит на открытый ключ."""
        return sum(b * a for b, a in zip(bits, self.public_seq))

    def decrypt_block(self, c: int) -> list[int]:
        """D(c): шаг 1 — S' = c · w⁻¹ mod q; шаг 2 — жадный алгоритм по B."""
        s_prime = (c * self.w_inv) % self.q
        bits = solve_superincreasing_knapsack(s_prime, self.private_seq)
        if bits is None:
            raise ValueError(f"Не удалось расшифровать c = {c}")
        return bits

    def verify_block(self, bits: list[int]) -> bool:
        """V(E(m), D(c)): проверяет D(E(m)) == m."""
        return self.decrypt_block(self.encrypt_block(bits)) == bits
```

## Связь кода с теорией

| Математическая операция | Строка кода | Пояснение |
|---|---|---|
| $b_i > \sum_{j=1}^{i-1} b_j$ | `next_val = total + random.randint(1, ...)` | Каждый элемент генерируется как `total + δ`, что гарантирует сверхвозрастаемость |
| $q > \sum_{i=1}^{n} b_i$ | `generate_prime_in_range(total + 1, ...)` | Модуль выбирается больше суммы B, чтобы при дешифровании не терять данные |
| $\gcd(w, q) = 1$ | `while gcd(self.w, self.q) != 1` | Взаимная простота обеспечивает существование $w^{-1}$ |
| $a_i = w \cdot b_i \mod q$ | `[(self.w * b) % self.q for b in self.private_seq]` | Модулярное умножение преобразует «лёгкий» рюкзак в «трудный» |
| $c = \sum m_i \cdot a_i$ | `sum(b * a for b, a in zip(bits, self.public_seq))` | Шифрование — скалярное произведение бит на открытый ключ |
| $S' = c \cdot w^{-1} \mod q$ | `s_prime = (c * self.w_inv) % self.q` | Дешифрование, шаг 1: переход от трудного к лёгкому рюкзаку |
| $w \cdot w^{-1} \equiv 1 \pmod{q}$ | `mod_inverse(self.w, self.q)` | Расширенный алгоритм Евклида находит обратный элемент |
| Жадный алгоритм: $b_i \leq S' \Rightarrow m_i = 1$ | `if sequence[i] <= remainder: bits[i] = 1` | Дешифрование, шаг 2: решение лёгкого рюкзака за O(n) |

### Пример пошагового выполнения

Пусть $B = (2, 5, 11, 23, 47, 95, 191, 383)$, $q = 881$, $w = 588$.

1. **Открытый ключ:** $a_i = 588 \cdot b_i \mod 881$
   - $a_1 = 588 \cdot 2 \mod 881 = 295$
   - $a_2 = 588 \cdot 5 \mod 881 = 177$
   - и т.д.

2. **Шифрование** $m = (1, 0, 1, 1, 0, 0, 1, 0)$:
   - $c = 1 \cdot a_1 + 0 \cdot a_2 + 1 \cdot a_3 + 1 \cdot a_4 + 0 \cdot a_5 + 0 \cdot a_6 + 1 \cdot a_7 + 0 \cdot a_8$

3. **Дешифрование:**
   - $w^{-1} = 588^{-1} \mod 881$ (вычисляется через расширенный алгоритм Евклида)
   - $S' = c \cdot w^{-1} \mod 881$
   - Жадный алгоритм по $B$: восстанавливаем биты
