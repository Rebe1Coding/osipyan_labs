# Алгоритм классического мультипликативного рюкзака (МВКР)

## Идея метода (схематично)

Криптосистема мультипликативного рюкзака (МВКР) основана на задаче о произведении подмножеств (Subset Product Problem). Идея аналогична аддитивному рюкзаку (АВКР), но вместо суммы используется произведение, а вместо модулярного умножения — модулярное возведение в степень.

**Ключевое отличие от АВКР:**
- АВКР: закрытый ключ — сверхвозрастающая последовательность, маскировка — умножение на `w mod q`
- МВКР: закрытый ключ — различные простые числа, маскировка — возведение в степень `w mod q`

```
Генерация ключей:
  B (различные простые) ──[pi^w mod q]──> A (открытый ключ)
       закрытый ключ                     выглядит как случайная
                                         последовательность

Шифрование (отправитель знает только A):
  m = (m₁, m₂, ..., mₙ) ──[c = ∏ aᵢ^mᵢ mod q]──> c (шифротекст)

Дешифрование (получатель знает B, q, w):
  c ──[P = c^{w⁻¹} mod q]──> P ──[факторизация по B]──> m
```

**Почему это работает:** набор различных простых чисел — это частный случай, для которого задача о произведении решается тривиально: нужно лишь проверить делимость на каждое простое число (основная теорема арифметики гарантирует единственность). Возведение в степень `w mod q` «маскирует» простую структуру, превращая задачу в NP-трудную для стороннего наблюдателя. Обладатель секрета `w` может «снять маску», возведя шифротекст в степень `w⁻¹ mod (q-1)`.

## NP-сложная задача в основе МВКР

В основе МВКР лежит **задача о произведении подмножеств** (Subset Product Problem, SPP):

> Дан набор положительных целых чисел $\{a_1, a_2, \ldots, a_n\}$ и целое число $P$. Требуется определить, существует ли подмножество $I \subseteq \{1, 2, \ldots, n\}$, такое что $\prod_{i \in I} a_i \equiv P \pmod{q}$.

Эта задача является **NP-полной** — она сводится к задаче о сумме подмножеств (SSP) через логарифмирование. Для произвольного набора чисел наилучшие известные алгоритмы:

- **Полный перебор:** $O(2^n)$
- **Meet-in-the-middle:** $O(2^{n/2})$
- **Связь с дискретным логарифмом:** восстановление закрытого ключа из открытого требует решения задачи DLP

## Криптостойкость МВКР

Криптостойкость МВКР основана на двух вычислительно трудных задачах:

1. **Задача о произведении подмножеств (SPP):** зная открытый ключ $A$ и шифротекст $c$, найти биты $m_i$ такие, что $\prod a_i^{m_i} \equiv c \pmod{q}$ — NP-полная задача.

2. **Задача дискретного логарифма (DLP):** зная $a_i = p_i^w \bmod q$ и $q$, восстановить $w$ — задача, для которой неизвестен полиномиальный алгоритм (в общем случае).

**Сравнение с АВКР:** В 1982 г. А. Шамир показал, что аддитивный рюкзак (АВКР) можно взломать за полиномиальное время с помощью LLL-алгоритма. МВКР обладает дополнительным уровнем защиты за счёт связи с задачей дискретного логарифма, однако для реального использования рекомендуются более современные криптосистемы.

## Математическое описание

### Модель алфавитной криптосистемы В.О. Осипяна

Криптосистема оформлена в виде кортежа:

$$\Sigma_D = (M^*, Q, C^*, E(m), D(c) \mid V(E(m), D(c)))$$

где:

| Обозначение | Описание |
|---|---|
| $M^*$ | Множество всех открытых текстов $m = (m_1, m_2, \ldots, m_n)$, $m_i \in \{0, 1\}$ |
| $Q$ | Алфавит $\{0, 1\}$ — числовые эквиваленты элементов сообщений |
| $C^*$ | Множество шифротекстов — элементы $\mathbb{Z}_q^*$ (произведения по модулю $q$) |
| $E(m)$ | Алгоритм шифрования (прямое преобразование) |
| $D(c)$ | Алгоритм дешифрования (обратное преобразование) |
| $V(E(m), D(c))$ | Верификация: $D(E(m)) = m$ для любого $m \in M^*$ |

Альтернативные обозначения: $E(m) = K_F(\text{inp } F_0)$, $D(c) = K_F(\text{ran } F_0)$ — ключевые функции шифрования и дешифрования соответственно.

### Генерация ключей

**Шаг 1.** Выбирается набор **различных простых чисел** $B = (p_1, p_2, \ldots, p_n)$:

$$\gcd(p_i, p_j) = 1 \quad \text{для всех } i \neq j$$

Различные простые автоматически попарно взаимно просты. Это «лёгкий» закрытый ключ — произведение любого подмножества однозначно определяется через факторизацию (основная теорема арифметики).

**Шаг 2.** Выбирается модуль $q$ — простое число, большее произведения всех элементов $B$:

$$q > \prod_{i=1}^{n} p_i$$

Это гарантирует, что при дешифровании произведение простых не редуцируется по модулю.

**Шаг 3.** Выбирается секретный показатель $w$, взаимно простой с $q - 1$:

$$\gcd(w, q - 1) = 1, \quad 2 \leq w < q - 1$$

Условие $\gcd(w, q-1) = 1$ (а не $\gcd(w, q) = 1$ как в АВКР!) необходимо, потому что обратный элемент $w^{-1}$ вычисляется по модулю $q - 1$ (через малую теорему Ферма).

**Шаг 4.** Формируется **открытый ключ** $A = (a_1, a_2, \ldots, a_n)$:

$$a_i = p_i^w \bmod q, \quad i = 1, 2, \ldots, n$$

**Закрытый ключ:** $(B, q, w)$. **Открытый ключ:** $A$.

### Шифрование $E(m)$

Для сообщения $m = (m_1, m_2, \ldots, m_n)$, $m_i \in \{0, 1\}$:

$$c = E(m) = \prod_{i=1}^{n} a_i^{m_i} \bmod q$$

При $m_i = 0$ множитель $a_i^0 = 1$ (элемент не участвует). При $m_i = 1$ множитель $a_i^1 = a_i$ (элемент включён).

### Дешифрование $D(c)$

**Шаг 1.** Вычисляем:

$$P = c^{w^{-1}} \bmod q$$

где $w^{-1}$ — мультипликативный обратный по модулю $q - 1$:

$$w \cdot w^{-1} \equiv 1 \pmod{q - 1}$$

Корректность (по малой теореме Ферма $a^{q-1} \equiv 1 \pmod{q}$ для $a \not\equiv 0$):

$$P = c^{w^{-1}} = \left(\prod p_i^{w \cdot m_i}\right)^{w^{-1}} = \prod p_i^{w \cdot m_i \cdot w^{-1}} = \prod p_i^{m_i} \pmod{q}$$

Поскольку $q > \prod p_i \geq \prod_{m_i=1} p_i$, результат точен (без редукции по модулю).

**Шаг 2.** Факторизация $P$ по известным простым $B$:

```
для i от 1 до n:
    если P делится на p_i:
        m_i = 1, P = P / p_i
    иначе:
        m_i = 0
если P == 1: расшифровка успешна
```

Это $O(n)$ — аналог жадного алгоритма в АВКР. Корректность гарантируется основной теоремой арифметики.

### Верификация $V(E(m), D(c))$

$$D(E(m)) = D\left(\prod a_i^{m_i} \bmod q\right) = m$$

для любого $m \in M^*$, что следует из:

$$\left(\prod a_i^{m_i}\right)^{w^{-1}} = \prod p_i^{m_i \cdot w \cdot w^{-1}} \equiv \prod p_i^{m_i} \pmod{q}$$

## Реализация (код)

```python
"""
Алгоритм классического мультипликативного рюкзака (МВКР).

Оформлено в соответствии с моделью алфавитной криптосистемы
В.О. Осипяна: Sigma_D = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))).
"""

from __future__ import annotations

import random
import sys
from dataclasses import dataclass, field
from math import gcd

from PyQt6.QtWidgets import (
    QApplication, QMainWindow, QWidget, QVBoxLayout, QHBoxLayout,
    QLabel, QLineEdit, QPushButton, QTextEdit, QGroupBox, QSpinBox,
    QMessageBox, QTabWidget,
)
from PyQt6.QtCore import Qt
from PyQt6.QtGui import QFont


# ---------------------------------------------------------------------------
#  Вспомогательные функции теории чисел
# ---------------------------------------------------------------------------

def extended_gcd(a: int, b: int) -> tuple[int, int, int]:
    """Расширенный алгоритм Евклида: a·x + b·y = gcd(a, b).

    В МВКР используется для нахождения w⁻¹ mod (q-1).
    Отличие от АВКР: обратный вычисляется по модулю q-1,
    а не по модулю q, потому что в основе лежит малая теорема
    Ферма: a^{q-1} ≡ 1 (mod q).
    """
    if a == 0:
        return b, 0, 1
    g, x, y = extended_gcd(b % a, a)
    return g, y - (b // a) * x, x


def mod_inverse(a: int, m: int) -> int:
    """Мультипликативный обратный: a⁻¹ mod m.

    В МВКР: w⁻¹ mod (q-1). Используется при дешифровании:
    P = c^{w⁻¹} mod q восстанавливает произведение простых.
    """
    g, x, _ = extended_gcd(a % m, m)
    if g != 1:
        raise ValueError(f"Обратный элемент не существует: gcd({a}, {m}) = {g} ≠ 1")
    return x % m


def is_prime_miller_rabin(n: int, k: int = 20) -> bool:
    """Тест Миллера-Рабина.

    Для МВКР нужны большие простые (q > ∏ pᵢ),
    поэтому используем вероятностный тест вместо
    перебора делителей.
    """
    if n < 2:
        return False
    if n < 4:
        return True
    if n % 2 == 0:
        return False
    r, d = 0, n - 1
    while d % 2 == 0:
        r += 1
        d //= 2
    for _ in range(k):
        a = random.randrange(2, n - 1)
        x = pow(a, d, n)
        if x == 1 or x == n - 1:
            continue
        for _ in range(r - 1):
            x = pow(x, 2, n)
            if x == n - 1:
                break
        else:
            return False
    return True


def generate_prime_above(low: int) -> int:
    """Генерирует ближайшее простое > low.

    q должно быть > ∏ pᵢ, чтобы произведение
    не редуцировалось при дешифровании.
    """
    candidate = low + 1 if low % 2 == 0 else low + 2
    if candidate % 2 == 0:
        candidate += 1
    while not is_prime_miller_rabin(candidate):
        candidate += 2
    return candidate


def generate_n_primes(n: int) -> list[int]:
    """Генерирует первые n простых чисел.

    Различные простые — «лёгкий» ключ МВКР:
    произведение подмножества однозначно определяет
    это подмножество (основная теорема арифметики).
    """
    primes = []
    candidate = 2
    while len(primes) < n:
        if is_prime_miller_rabin(candidate):
            primes.append(candidate)
        candidate += 1
    return primes


# ---------------------------------------------------------------------------
#  Решение «лёгкого» мультипликативного рюкзака
# ---------------------------------------------------------------------------

def solve_coprime_product_knapsack(target: int, primes: list[int]) -> list[int] | None:
    """Факторизация target по известным простым — O(n).

    Это аналог жадного алгоритма для сверхвозрастающей
    последовательности в АВКР. Для различных простых
    проверяем делимость target на каждое pᵢ.
    Корректность: основная теорема арифметики.
    """
    n = len(primes)
    bits = [0] * n
    remainder = target
    for i in range(n):
        if remainder % primes[i] == 0:
            bits[i] = 1
            remainder //= primes[i]
    return bits if remainder == 1 else None


# ---------------------------------------------------------------------------
#  Криптосистема МВКР
# ---------------------------------------------------------------------------

@dataclass
class MultiplicativeKnapsackCryptoSystem:
    """Σ₀ = (M*, Q, C*, E(m), D(c) | V(E(m), D(c))) на основе МВКР.

    Закрытый ключ: B (различные простые), q (модуль),
                   w (секретный показатель, gcd(w, q-1) = 1).
    Открытый ключ: A, где aᵢ = pᵢ^w mod q.

    E(m) = ∏ aᵢ^mᵢ mod q          — произведение по открытому ключу
    D(c) = factor(c^{w⁻¹} mod q, B) — переход к лёгкому рюкзаку
    """
    block_size: int
    private_seq: list[int] = field(default_factory=list)
    q: int = 0
    w: int = 0
    public_seq: list[int] = field(init=False, default_factory=list)
    w_inv: int = field(init=False, default=0)

    def __post_init__(self) -> None:
        # Генерация/валидация закрытых простых B
        if not self.private_seq:
            self.private_seq = generate_n_primes(self.block_size)

        # ∏ pᵢ — используется для выбора q
        product_b = 1
        for b in self.private_seq:
            product_b *= b

        # q > ∏ pᵢ — произведение не «обрезается» при дешифровании
        if self.q == 0:
            self.q = generate_prime_above(product_b)

        # w: gcd(w, q-1) = 1 — для существования w⁻¹ mod (q-1)
        if self.w == 0:
            while True:
                self.w = random.randint(2, self.q - 2)
                if gcd(self.w, self.q - 1) == 1:
                    break

        # w⁻¹ mod (q-1) — ключевой элемент дешифрования
        # По малой теореме Ферма: a^{w·w⁻¹} = a^{1+k(q-1)} = a (mod q)
        self.w_inv = mod_inverse(self.w, self.q - 1)

        # Открытый ключ: aᵢ = pᵢ^w mod q
        # Возведение в степень «маскирует» простую структуру
        self.public_seq = [pow(b, self.w, self.q) for b in self.private_seq]

    def encrypt_block(self, bits: list[int]) -> int:
        """E(m) = ∏ aᵢ^mᵢ mod q — произведение выбранных элементов."""
        c = 1
        for b, a in zip(bits, self.public_seq):
            if b:
                c = (c * a) % self.q
        return c

    def decrypt_block(self, c: int) -> list[int]:
        """D(c): P = c^{w⁻¹} mod q, затем факторизация P по B."""
        p = pow(c, self.w_inv, self.q)
        bits = solve_coprime_product_knapsack(p, self.private_seq)
        if bits is None:
            raise ValueError(f"Не удалось расшифровать c = {c}")
        return bits

    def verify_block(self, bits: list[int]) -> bool:
        """V(E(m), D(c)): D(E(m)) == m."""
        return self.decrypt_block(self.encrypt_block(bits)) == bits
```

## Связь кода с теорией

| Математическая операция | Строка кода | Пояснение |
|---|---|---|
| $B = (p_1, \ldots, p_n)$ — различные простые | `generate_n_primes(self.block_size)` | Первые $n$ простых чисел: каждое попарно взаимно просто, что делает факторизацию тривиальной |
| $q > \prod_{i=1}^{n} p_i$ | `generate_prime_above(product_b)` | Модуль больше произведения всех элементов B, чтобы результат дешифрования был точным |
| $\gcd(w, q-1) = 1$ | `if gcd(self.w, self.q - 1) == 1` | Взаимная простота с $q-1$ (не с $q$!), потому что обратный берётся по модулю $q-1$ через малую теорему Ферма |
| $a_i = p_i^w \bmod q$ | `pow(b, self.w, self.q)` | Модулярное возведение в степень «маскирует» простые числа, создавая «трудную» последовательность |
| $c = \prod a_i^{m_i} \bmod q$ | `c = (c * a) % self.q` (в цикле) | Шифрование — произведение выбранных элементов открытого ключа по модулю $q$ |
| $P = c^{w^{-1}} \bmod q$ | `p = pow(c, self.w_inv, self.q)` | Дешифрование, шаг 1: «снятие маскировки» — переход от трудного к лёгкому рюкзаку |
| $w \cdot w^{-1} \equiv 1 \pmod{q-1}$ | `mod_inverse(self.w, self.q - 1)` | Расширенный алгоритм Евклида находит обратный показатель |
| Факторизация: $P \div p_i \Rightarrow m_i = 1$ | `if remainder % primes[i] == 0: bits[i] = 1` | Дешифрование, шаг 2: «лёгкий» рюкзак — проверка делимости за $O(n)$ |

### Пример пошагового выполнения

Пусть $B = (2, 3, 5, 7, 11, 13, 17, 19)$, $q = 9699713$ (простое, $> \prod B = 9699690$).

Допустим, $w = 7483449$, $w^{-1} \bmod (q-1) = 9630601$.

1. **Открытый ключ:** $a_i = p_i^w \bmod q$
   - $a_1 = 2^{7483449} \bmod 9699713$
   - $a_2 = 3^{7483449} \bmod 9699713$
   - и т.д. (вычисляется через быстрое возведение в степень)

2. **Шифрование** $m = (1, 0, 1, 1, 0, 0, 1, 0)$:
   - $c = a_1 \cdot a_3 \cdot a_4 \cdot a_7 \bmod q$
   - Выбраны элементы с индексами, где $m_i = 1$

3. **Дешифрование:**
   - $P = c^{w^{-1}} \bmod q = c^{9630601} \bmod 9699713$
   - $P = 2 \cdot 5 \cdot 7 \cdot 17 = 1190$ (точное произведение, $P < q$)
   - Факторизация: $1190 \div 2 = 595$, $595 \div 5 = 119$, $119 \div 7 = 17$, $17 \div 17 = 1$
   - $m = (1, 0, 1, 1, 0, 0, 1, 0)$ — исходное сообщение восстановлено

### Сравнение МВКР и АВКР

| Характеристика | АВКР (аддитивный) | МВКР (мультипликативный) |
|---|---|---|
| Операция шифрования | Сумма: $c = \sum m_i \cdot a_i$ | Произведение: $c = \prod a_i^{m_i} \bmod q$ |
| Закрытый ключ | Сверхвозрастающая последовательность | Различные простые числа |
| Маскировка | $a_i = w \cdot b_i \bmod q$ | $a_i = p_i^w \bmod q$ |
| Обратное преобразование | $S' = c \cdot w^{-1} \bmod q$ | $P = c^{w^{-1}} \bmod q$ |
| Решение лёгкого рюкзака | Жадный алгоритм (справа налево) | Факторизация по известным простым |
| Модуль обратного | $w^{-1} \bmod q$ | $w^{-1} \bmod (q-1)$ |
| Связанная трудная задача | Subset Sum (NP-полная) | Subset Product + DLP |
